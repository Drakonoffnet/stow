# Stow — бриф для дизайнера

> Опис реалізованого функціоналу, поточний UI, стани та обмеження.
> Мета: щоб дизайнер міг підготувати макети, які **реально втілюються** в поточному стеку.
> Версія застосунку: 0.1.0 (MVP). Платформа: macOS.

---

## 1. Що це

**Stow** — десктоп-застосунок для macOS, який **паралельно пакує теки в zip і відправляє**
готові архіви в місце призначення: локальна/мережева тека або віддалений сервер по SSH/SFTP.

Сценарій: користувач кидає теки у вікно (або обирає через діалог), вказує призначення,
натискає **Start** — кожна тека пакується в окремий архів і переміщується в призначення;
прогрес і результат видно в списку задач.

Один екран, одне вікно. Без онбордингу, без акаунтів.

---

## 2. Реалізований функціонал

| Можливість | Деталі |
|---|---|
| Джерело (Source) | Кілька тек одночасно. Додавання: drag-and-drop або кнопка вибору. Видалення окремих елементів і очищення всього списку. |
| Призначення (Destination) | Перемикач **Local** / **SSH**. Local — тека (drag-and-drop або діалог). SSH — форма підключення. |
| Пакування | Кожна тека → окремий `.zip`. Паралельне стискання файлів (кількість воркерів = число ядер CPU). |
| Прогрес | По кожній задачі: лічильник файлів і прогрес-бар; окремий стан перенесення. |
| Скасування | Кнопка скасування активної задачі; очищення завершених зі списку. |
| Опції | «Видалити джерело після успіху», «порахувати sha256». |
| SSH/SFTP | Авторизація: SSH-агент / пароль / ключ. Секрети зберігаються в macOS Keychain. |
| Лог | Текстовий журнал подій унизу вікна. |

**Ще не реалізовано (поза поточним обсягом):** хмара S3; пауза/відновлення задач;
черга з пріоритетами; історія між запусками; багатомовність (UI лише англійською).

## 3. Поточний UI-інвентар

Усі підписи зараз **англійською** (це база, з якої працюємо).

### Вікно
- Заголовок: `Stow — pack and move`.
- Розмір за замовчуванням: **760 × 640**, мінімальний: **560 × 480**. Вікно змінюється в розмірі.
- Заголовок секції зверху: `Pack folders into zip and move`.

### Дві колонки
**Колонка 1 — Source**
- Зона drag-and-drop із заголовком `Source` і підказкою `Drop folders to pack here` (порожній стан).
- Кожен елемент списку: кнопка `✕` (видалити) + `📁 <ім'я теки>`.
- Кнопка `➕ Add folders…` (нативний діалог, множинний вибір).

**Колонка 2 — Destination**
- Перемикач: `Local` / `SSH`.
- **Local:** зона drag-and-drop `Destination` / `Drop the folder for archives here`; кнопки `📂 Choose folder…` та `Clear`.
- **SSH:** форма — поля `Host`, `Port`, `User`, `Remote dir`; перемикач `Auth: Agent / Password / Key`;
  залежно від вибору — поле `Password` або `Key file` + `Passphrase` (поля секретів масковані).

### Панель дій
- Чекбокси: `Remove source after success`, `sha256`.
- Кнопки: `▶ Start` (активна лише коли є джерела й валідне призначення), `Clear sources`.

### Список задач (Jobs)
- Заголовок `Jobs:` + кнопка `Clear finished`.
- Порожній стан: `no jobs yet`.
- Рядок задачі: назва теки + ідентифікатор (`#1`) + кнопка `Cancel` (лише для активних).
- Статуси (див. §4).

### Лог (Log)
- Заголовок `Log:` + список рядків моноширинним шрифтом, автопрокрутка донизу.

## 4. Стани, які треба покрити в дизайні

### Життєвий цикл задачі
```
Queued → Archiving(done/total) → Transferring → Done
                 └──────────────→ Failed
                 └──────────────→ Canceled
```
- `queued…` — у черзі.
- `packing N/M` — прогрес-бар стискання (частка = N/M).
- `moving…` — перенесення (прогрес-бар майже повний, невизначений).
- `✔ done: <шлях>` (+ рядок `sha256: …`, якщо ввімкнено) — успіх.
- `✘ <текст помилки>` — помилка.
- `canceled` — скасовано.

### Стани зон і форм
- **Зона drag-and-drop:** звичайна / підсвічена (коли над нею тягнуть файли) / з елементами / порожня.
- **Кнопка Start:** активна / неактивна (немає джерел або призначення).
- **SSH-форма:** три варіанти авторизації змінюють набір полів.
- **Помилка призначення / валідації:** наразі виводиться рядком у лог (можна підняти в окремий банер — на розсуд дизайну).

### Поточна семантика кольору (можна замінити в палітрі)
- Акцент/підсвітка зони: синій `#5AAAFF`.
- Успіх: зелений `#50C878`.
- Помилка: червоний `#E66464`.
- Іконка застосунку: синьо-ціановий градієнт (`#3B82F6 → #06B6D4`), деталі — `assets/icon.svg`.

## 5. Технічні обмеження (впливають на дизайн)

### Стек UI — це найважливіше
UI побудований на **egui** (immediate-mode GUI на Rust). Це задає рамки того, що **реально втілити**:

**Можна:**
- Тема light/dark, акцентний колір, відступи (spacing), радіуси заокруглення, товщина рамок.
- Власні шрифти (важливо: для не-латиниці треба підключити шрифт із потрібними гліфами).
- Базові компоненти: кнопки, чекбокси, радіо/перемикачі, текстові поля, прогрес-бари, скрол-зони, картки (frames), роздільники, підказки (tooltips).
- Емодзі-іконки (як зараз: `📁 ✕ ➕ 📂 ▶ ✔ ✘`) або вбудовані растрові/SVG-іконки.

**Складно / небажано:**
- Піксель-перфект із складними накладаннями, довільні градієнти всередині віджетів, складні тіні.
- Багаті анімації та переходи (egui має обмежену анімацію).
- Нестандартні розкладки поза сіткою/колонками/рядками.

➡️ **Висновок для дизайну:** робити **плаский, компонентний стиль** (як macOS-native / Apple HIG-friendly),
тему світлу й темну, чітку сітку. Уникати ефектів, які важко відтворити в immediate-mode.

### Функціональні / платформні обмеження
- **ОС:** macOS 11+; зараз збірка arm64 (universal arm64+x86_64 опційно).
- **Одне вікно**, без вкладок і кількох вікон.
- **Формат архіву:** стандартний zip, Deflate рівень 6.
  - Без ZIP64 → файли й архів **до 4 ГБ**.
  - **Порожні теки не зберігаються** в архіві.
  - Імена у UTF-8: Finder/`ditto` відкривають коректно; **старий термінальний `unzip` спотворює кирилицю**.
- **Іменування:** `<ім'я_теки>_<РРРР-ММ-ДД_ГГХХСС>.zip`, **перезапис заборонено** (колізій майже немає завдяки часовій мітці).
- **Атомарність:** архів з'являється в призначенні лише цілим (тимчасовий файл → перейменування).
- **Паралельність:** кілька задач одночасно + паралельне стискання файлів усередині задачі.
- **SSH/SFTP:** автентифікація агентом/паролем/ключем; секрети — у **macOS Keychain**;
  перевірка host-key за `~/.ssh/known_hosts` (невідомий хост приймається при першому з'єднанні).
- **Безпека дистрибуції:** ad-hoc підпис; **не нотаризовано** (для роздачі іншим — окремий крок).

## 6. Що очікується від дизайнера

1. **Макети екрана** для всіх станів §4 (порожній / з даними / прогрес / успіх / помилка / скасовано),
   у **світлій і темній** темах.
2. **Палітра** (акцент, успіх, помилка, нейтральні відтінки фону/тексту) під light/dark.
3. **Шкала відступів і типографіка** (розміри, накреслення). Якщо UI стане багатомовним —
   врахувати розширення тексту та **шрифт із кирилицею**.
4. **Компоненти/стани:** кнопки (звич./hover/disabled), чекбокси, перемикачі, поля (звич./focus/масковані),
   прогрес-бар, картка-зона drag-and-drop (звич./підсвічена), рядок задачі, рядок логу.
5. **Іконки** в стилі застосунку (можна як SVG/PNG для вбудовування або підбір емодзі-набору).
   Іконка застосунку вже є (`assets/icon.svg`) — бажано тримати єдиний стиль.
6. **Адаптив:** поведінка двох колонок при мінімальній ширині (560 px) — складати в один стовпчик чи стискати.
7. *(Опційно)* пропозиція винести помилки валідації/призначення в окремий **банер/inline-повідомлення**
   замість лише логу.

### Поза обсягом дизайну (поки що)
- Екрани S3, історія задач, налаштування з багатьма сторінками, онбординг, кілька вікон.
